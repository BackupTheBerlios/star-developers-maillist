> Star can detect lzip-compressed files and automatically decompress them.

Please check the preliminary source I just put out at:

ftp://ftp.berlios.de/pub/schily/

your patch contained a typo.

> Lzip is another file format which uses LZMA compression, in some ways a
> "competitor" of xz. See http://lzip.nongnu.org/lzip.html
>
> Some lzip-compressed tar files can be downloaded from
> http://download.savannah.gnu.org/releases/lzip/

Did you make tests with this compression program?

My impression was that it spends aprox. twice as much CPU time than xz for the 
same compression task, OK xz needs twice as much RAM than lzip.

The decompression time seems to be comparable and lzip again needs only half 
the RAM xz like to have, so this may be an argument. The licensce of lzip is a 
problem as it is unclear whether the code may be used as a built in library for 
arbitrary projects.

The interesting thing seems to be that both xz and lzip claim to be a 
descendent frm LZMA. Do you have more information on this?
