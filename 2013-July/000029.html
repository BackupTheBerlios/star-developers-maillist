Another thing I found when observing UID/GID restoration under root is
that the ownership is _always_ restored when using 'pax' API (same as in
star and other frontends).

According to 'pax' specification (and the -p option), I would guess this
is bad behaviour because when things are done this way — root user is
unable to restore archive without also restoring the original ownership of
archived files.
