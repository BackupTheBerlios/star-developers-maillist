maybe I now understand your problem.

Are your problems caused by a script that launches star with euid != uid?

Star asumes that if euid != uid, then it has been installed suid root and it 
needs to reset euid to uid before it can continue. Star allows to be installed 
suid root to be able to establish a fast network connetion (one that does not 
pipe to "rsh") to a remote tape server.

BTW: I did switch Schillix-ON a OpenSolaris continuation project to use spax 
instead of the closed source opengroup pax. I am definitely interested to have 
star being POSIX compliant.
